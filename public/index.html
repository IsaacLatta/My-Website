CGI-HTTPS/HTTP Web Server

This custom-built CGI web server demonstrates a deep understanding of HTTP, security, and configuration-based design. It handles GET, HEAD, and POST methods; executes dynamic scripts via POSIX calls; and provides optional SSL (HTTPS) support. Role-based access control is managed through JWT tokens, with hierarchical permissions defined in an XML configuration. Featureing modular middleware pipeline for logging, routing, authentication, and error handling.

HTTP/HTTPS Reverse Proxy Server

A fully developed reverse proxy server using standalone Asio, this project forwards TCP and HTTP requests to defined back-end servers, optionally over SSL. The application logs traffic and can run in the background (daemonized) or handle requests interactively. It also features a command-line interface for port and address configuration, plus optional HTTPS handling for encrypted connections.

This Web Server

Technical Description

The web content you are seeing right now is currently being served from my custom web server.

This entire website is powered by my custom-built CGI server written in C++ using ASIO for networking, OpenSSL for HTTPS, and jwt-cpp for authentication. The server reads its configuration from an XML file, while Python scripts handle dynamic or actionable features (such as contact forms and other back-end logic). At the moment, it’s running on physical hardware in my home environment, although I may move it to a cloud-based solution in the future.

All incoming traffic first passes through a Web Application Firewall (WAF) and is then routed via a series of proxies (either nginx or my own load balancer, depending on the setup at the time). I’ve also bolstered security with a firewall, EDR software, and in-house scripts that manage both the proxy processes and Docker images hosting these services.

If you’d like to see how everything works behind the scenes—including the server code, scripts and front-end code—please feel free to check out my GitHub. I’ve omitted some sensitive details for security reasons, but the source code and documentation provide a comprehensive look at the system’s design and capabilities.
